---
title: "Update Order"
api: "PUT /api/pedidos/{id}"
---

Updates an existing order. Typically used to change order status or modify line items.

## Authentication

Requires JWT token in the Authorization header.

```bash
Authorization: Bearer YOUR_JWT_TOKEN
```

## Path Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `id` | Long | Yes | Order ID to update |

## Request Body

Send a complete `PedidoDetalleDTO` object with updated values.

```json
{
  "usuario": {
    "usuario_id": 5
  },
  "estado": "ENVIADO",
  "detalles": [
    {
      "detalle_id": 101,
      "producto": {
        "producto_id": 10
      },
      "cantidad": 2,
      "precioUnitario": 49.99
    },
    {
      "detalle_id": 102,
      "producto": {
        "producto_id": 15
      },
      "cantidad": 1,
      "precioUnitario": 129.99
    }
  ]
}
```

### Request Fields

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `usuario.usuario_id` | Long | Yes | User ID |
| `estado` | String | Yes | Updated order status |
| `detalles` | Array | Yes | Array of order line items |
| `detalles[].detalle_id` | Long | No | Line item ID (for existing items) |
| `detalles[].producto.producto_id` | Long | Yes | Product ID |
| `detalles[].cantidad` | Integer | Yes | Quantity |
| `detalles[].precioUnitario` | BigDecimal | Yes | Unit price |

### EstadoPedido Values

- `PENDIENTE` - Order pending
- `PAGADO` - Order paid
- `ENVIADO` - Order shipped
- `ENTREGADO` - Order delivered
- `CANCELADO` - Order cancelled

## Example Request

```bash
curl -X PUT "https://api.iquea.com/api/pedidos/42" \
  -H "Authorization: Bearer YOUR_JWT_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "usuario": {
      "usuario_id": 5
    },
    "estado": "ENVIADO",
    "detalles": [
      {
        "detalle_id": 101,
        "producto": {
          "producto_id": 10
        },
        "cantidad": 2,
        "precioUnitario": 49.99
      }
    ]
  }'
```

## Status Codes

| Code | Description |
|------|-------------|
| 200 | Success - Order updated |
| 400 | Bad request - Invalid order data |
| 401 | Unauthorized - Invalid or missing token |
| 404 | Order not found |
| 500 | Internal server error |
